# 分支的使用

## 创建和切换分支
创建
```
git branch dev
```
切换
```
git checkout dev
```
创建并切换
```
git checkout -b dev
```
**切换分支后看到的工作区和当前分支一致**

## 查看当前分支
打星号的是当前分支
```
git branch
```

## 合并到master分支
```
git merge dev
```

## 删除分支
```
git branch -d dev
```

## 解决分支冲突
merge指令出问题，直接改一个文件就可以。
必须要有不一样才会冲突，只是包含关系可以合并。

## 保留原来分支
开发中master可以作为稳定版本，dev作为修改的中间过程。
所以在dev合并时可以用以下语句，保证dev的保留并对合并后的分支进行命名。
```
git merge --no-ff -m "Branch" dev
```
